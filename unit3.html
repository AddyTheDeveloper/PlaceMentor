    1. SQL Mastery: Categories & Commands

    CategoryFull FormKey Commands
    DDLData Definition LanguageCREATE, ALTER, DROP, TRUNCATE
    DMLData Manipulation LanguageSELECT, INSERT, UPDATE, DELETE
    DCLData Control LanguageGRANT, REVOKE
    TCLTransaction Control LanguageCOMMIT, ROLLBACK, SAVEPOINT

    Delete vs Truncate vs Drop:
    Delete: DML, removes rows, can be rollbacked.
    Truncate: DDL, removes all rows, cannot be rollbacked, faster.
    Drop: DDL, removes whole table structure.

    2. Database Normalization (Step-by-Step)

    1NF (First Normal Form): Eliminate duplicate columns. No multi-valued attributes (atomicity).
    2NF (Second Normal Form): Should be in 1NF + No partial functional dependency (every non-key attr must depend on WHOLE primary key).
    3NF (Third Normal Form): Should be in 2NF + No transitive dependency (non-key attr shouldn't depend on another non-key attr).
    BCNF (Boyce-Codd NF): Stronger than 3NF. For every FD X → Y, X must be a super key.

    RDBMS: Relational Database Management System. Data is stored in tables (Relations) with pre-defined schemas.

    ACID Properties (The Gold Standard)
    • Atomicity: All or nothing execution.
    • Consistency: DB stays in valid state before and after transaction.
    • Isolation: Transactions don't interfere.
      Levels: Read Uncommitted, Read Committed, Repeatable Read, Serializable.
    • Durability: Changes are permanent even after system failure (WAL).

    3. Normalization (Data Hygiene)

    Normal Form Check: Always identify the Primary Key (PK) first, then identify Functional Dependencies (FDs).

    3. ACID Properties & Transactions

    Transaction Isolation Levels (SQL Standard):
    1. Read Uncommitted: Lowest; allows dirty reads.
    2. Read Committed: No dirty reads; allows non-repeatable reads.
    3. Repeatable Read: No non-repeatable reads; allows phantom reads.
    4. Serializable: Highest; strict isolation.

    • Atomicity: Transaction is all-or-nothing.
    • Consistency: Database must go from one valid state to another.
    • Isolation: Transactions don't interfere.
    • Durability: Changes are permanent.

    3. Concurrency Control & 2PL

    Two-Phase Locking (2PL): Ensures serializability.
    1. Growing Phase: Transaction only obtains locks.
    2. Shrinking Phase: Transaction only releases locks.

    4. Query Optimization

    The Optimizer chooses the "Execution Plan" with the minimum cost.
    Heuristic-based: Follows rules like "Perform selections before joins."

    Normalization: "How to Solve" Example

    Table R(A, B, C, D) with FDs: {A → B, B → C}.

    1. Candidate Key: A (since A+= {A,B,C,D} if we assume D is trivial or part of a key). If A → ABCD, A is CK.
    2. 1NF: All attributes atomic.
    3. 2NF: No partial dependency. (Since CK is single attr A, it's already 2NF).
    4. 3NF: No transitive dependency. Here A → B → C (Transitive).
      Fix: Decompose into R1(A, B) and R2(B, C).

    3. Indexing & B+ Trees

    Indexing: Data structure technique to quickly retrieve records. B+ Trees are preferred over B-Trees in DBMS because data is only stored in leaf nodes, allowing faster range queries.
