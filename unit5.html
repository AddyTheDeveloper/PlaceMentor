    1. Linked Lists (Linear Dynamics)

    TypeStructureKey Characteristic
    SinglyData → NextSimple. Uni-directional.
    DoublyPrev ↔ Data ↔ NextBi-directional traversal; Uses more memory.
    CircularLast → FirstInfinite traversal; Used in Round-Robin scheduling.

    Floyd's Cycle Finding (Fast & Slow):

    1. Start slow and fast at HEAD.
    2. Move slow by 1 step, fast by 2 steps.
    3. If slow == fast, a cycle exists!
    4. Why? The distance between them increases by 1 each step. In a loop, they will eventually occupy the same position.

    2. Stacks & Queues (Order Patterns)

    Stack (LIFO)

    Apps: Recursion, Undo/Redo, Balanced Parentheses.

    Queue (FIFO)

    Apps: BFS, Print Spooler, CPU Job Scheduling.

    Circular Queue: Solves the limitation of linear queue where space is wasted even if FRONT is not zero.

    BST Search: O(h) where h is height.
    AVL Search: O(log n) guaranteed because it balances.

    4. Trie (Prefix Tree)

    Trie: Used for fast retrieval of strings/prefixes. Used in Auto-complete, Spell checkers.
    Search/Insert: O(L) where L is length of string.

    5. Disjoint Set Union (DSU)

    Purpose: Maintain sets of elements and merge them.
    Union by Rank: Keeps tree height small.
    Path Compression: Flattens tree during find operation. Combined complexity is nearly constant O(α(n)).

    3. Trees: The Hierarchical Power

    BST (Binary Search Tree): Left child < Parent < Right child.

    Balanced Trees (Interview Gold)

    • AVL Trees: Height-balanced BST where diff in heights of left/right subtrees is ≤ 1.
    • Red-Black Trees: Uses color coding for self-balancing. Used in std::map in C++.
    RotationWhen to use?
    LL / RRSingle rotation for simple imbalance.
    LR / RLDouble rotation for zig-zag imbalance.
    Placement Case Study: Map vs BST

    Question: "When would you prefer a Treemap (BST) over a Hashmap?"

    Answer: When order matters or you need to perform range queries (e.g., "Find all keys between 10 and 50"). Hashmaps have O(1) average lookup but no inherent order.

    4. Graphs & Hashing Collision

    Graph: G = (V, E) where V=Vertices, E=Edges.

    BFS: Queue based. Finds shortest path in unweighted graph.
    DFS: Stack/Recursion based. Used for cycle detection.

    Hashed Conflict Resolution

    TechniqueDescription
    ChainingUses Linked List at each index. No overflow issue.
    Open AddressingFinds next available slot (Linear Probing, Quadratic Probing).
    Graph Algorithms: Shortest Paths

    Dijkstra: O(E log V). Greedy. Doesn't work for negative edges.
    Bellman-Ford: O(V*E). DP. Works with negative edges and detects negative cycles.

    Consistent Hashing: Used in distributed systems (like Load Balancers) so that when a node is added/removed, only K/n keys need to be remapped.
