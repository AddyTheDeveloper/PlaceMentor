    • Unit 1: Operating Systems

    The Core Bridge: Hardware Management, Process Control, and Memory Architecture.

    1. Foundations & Types of OS

    Technical Definition: An Operating System (OS) is a system software that manages computer hardware, software resources, and provides common services for computer programs. It acts as an intermediary between the user and the hardware.

    Functions of Operating System

    Resource Management: Allocates CPU, memory, and I/O devices to processes.
    File Management: Handles file creation, deletion, and storage organization.
    Security: Protects data and resources from unauthorized access.
    Error Detection: Monitors the system for hardware/software malfunctions.

    Kernel vs User Mode

    Modern OSs use a hardware bit to switch between two modes of operation to protect the system.

    FeatureUser ModeKernel Mode
    PrivilegeLow Privilege (Restricted Access)High Privilege (Full Access)
    ExecutionUser applications run here.Core OS services/Kernel run here.
    Direct HW AccessNot allowed (must use System Calls).Direct access to all hardware.
    Bit ValueUsually 1Usually 0
    Crash ImpactOnly the application crashes.Entire system may crash (BSOD).

    Architecture: Monolithic vs. Microkernel

    Monolithic Kernel

    All OS services (VFS, IPC, File System, Scheduler) run in kernel space. Fast due to low overhead, but one failure can crash the whole system.

    Pros: Performance, Simple communication.
    Cons: Hard to debug, Low security.

    Microkernel

    Only essential services (IPC, Address Space) stay in kernel. Others move to user-space as "Servers". Highly stable and secure.

    Pros: Modular, Secure, Flexible.
    Cons: Slower due to message passing overhead.

    2. Process Management & Life Cycle

    Process vs Program: A program is a 'passive' entity (executable file on disk). A process is an 'active' entity (program in execution, includes PC, stack, data section).

    Process Life Cycle (State Diagram)

    Process Control Block (PCB)

    The "ID Card" of a process. Every process has its own PCB containing:

    • Process ID (PID)
    • Process State
    • Program Counter (PC)
    • Registers
    • Memory Limits
    • Open Files List

    Context Switching

    When the CPU switches from one process to another, the OS must save the state of the old process and load the saved state of the new process. This is called **Context Switching**.

    Overhead: Context switching is pure overhead—no useful work is done while switching. Hardware support (like multiple register sets) can speed it up.

    Threads: Lightweight Processes

    User-Level Threads
    • Managed by Thread Library.
    • Kernel is unaware of them.
    • Fast creation/switching.
    • If one thread blocks, all block.
    Kernel-Level Threads
    • Managed by OS Kernel.
    • Kernel schedules them.
    • Slower creation/switching.
    • Individual blocking allowed.
    Multithreading Models:

    Many-to-One: Many user threads mapped to one kernel thread.
    One-to-One: Each user thread mapped to its own kernel thread (e.g., Windows, Linux).
    Many-to-Many: Many user threads mapped to a smaller or equal number of kernel threads.

    3. Processor Scheduling Algorithms

    Burst Time (BT): Time required by process for CPU execution.
    Arrival Time (AT): Time at which process enters Ready Queue.

    Turnaround Time (TAT) = Exit Time - Arrival Time

    Waiting Time (WT) = TAT - Burst Time

    Common Scheduling Algorithms

    AlgorithmTypeCriteriaKey Facts
    FCFSNon-PreemptiveArrival TimeSimple but suffers from Convoy Effect.
    SJFNon-PreemptiveBurst TimeGives minimum average waiting time. Difficult to predict BT.
    SRTFPreemptiveRemaining TimePreemptive version of SJF.
    Round RobinPreemptiveTime QuantumBest for Time-Sharing. No Starvation.
    PriorityBothPriority NumberHigher priority runs first. Can cause Starvation.

    Solution to Starvation: "Aging" is a technique of gradually increasing the priority of processes that wait in the system for a long time.

    4. Process Synchronization & IPC

    Race Condition: A situation where the final outcome depends on the order of execution of multiple processes accessing shared data.

    Critical Section Problem Requirements:

    1. Mutual Exclusion: Only one process at a time.
    2. Progress: Decisions on who enters next aren't delayed indefinitely.
    3. Bounded Waiting: No process waits forever to enter.

    Synchronization Tools

    Mutex (Mutual Exclusion)
    • Locking mechanism.
    • Only one thread can hold the lock.
    • Binary state (Locked/Unlocked).
    • Owned by the thread that locked it.
    Semaphore
    • Signaling mechanism.
    • Binary (0,1) or Counting (0 to N).
    • Operations: wait() [P] and signal() [V].
    • No ownership concept.

    Classical Problems of Synchronization

    • Producer-Consumer: Buffer size management between producer and consumer.
    • Dining Philosophers: Dealing with resource allocation and deadlocks.
    • Readers-Writers: Multiple readers allowed, but only one writer.

    4b. Deadlocks (The Infinite Wait)

    Deadlock: A cyclic dependency where each process in a set is waiting for a resource held by another process in that same set.

    Necessary Conditions (Coffman Conditions)
    1. Mutual Exclusion: Non-shareable resources.
    2. Hold and Wait: Holding at least one resource while waiting for others.
    3. No Preemption: Resources can't be taken away by force.
    4. Circular Wait: P1 → P2 → ... → Pn → P1 chain of waiting.

    Deadlock Handling Strategies

    StrategyApproach
    PreventionEnsure at least one Coffman condition never happens.
    AvoidanceSystem checks if allocation leads to "Safe State" (e.g., Banker's Algorithm).
    Detection & RecoveryAllow deadlock, detect it using RAG (Resource Allocation Graph), then kill/preempt.
    Ignorance"Ostrich Algorithm" - pretend it never happens (Default in Windows/Linux).
    Banker's Algorithm: A Closer Look

    Goal: Find if there exists a **Safe Sequence** where everyone can finish.

    Data Structures: Available (Vector), Max (Matrix), Allocation (Matrix), Need (Matrix).

    Formula: Need[i,j] = Max[i,j] - Allocation[i,j]

    Deadlock vs Starvation: Deadlock is a circular wait where no one progresses. Starvation is where one process waits indefinitely while others use resources (Priority-based).

    4c. IPC (Inter-Process Communication)

    Processes need to communicate to share data or synchronize.

    Shared Memory

    Fastest. Direct memory access. Programmer must manage synchronization (Mutex/Semaphores).

    Message Passing

    Slower. Kernel manages communication. No sync needed. Ideal for distributed systems.

    Examples: Pipes, Sockets, Signals, Message Queues.

    5. Memory Management

    Memory Management: Handling the coordination of RAM, ensuring processes have enough space while optimizing utilization and protection.

    Logical vs Physical Address

    • Logical Address: Generated by the CPU (Virtual Address).
    • Physical Address: Actual address in the RAM unit.
    • MMU (Memory Management Unit): Hardware device that maps logical address to physical address at runtime.

    Paging vs Segmentation

    FeaturePagingSegmentation
    Block SizeFixed-size blocks (Pages/Frames).Variable-size blocks (Segments).
    FragmentationSuffer from Internal Fragmentation.Suffer from External Fragmentation.
    User ViewOS-centric (Splits code into fixed units).User-centric (Functions, Stack, Main).
    VisibilityInvisible to programmer.Visible to programmer.
    TLB & Page Faults

    TLB (Translation Lookaside Buffer): A cache for page table entries. Reduces memory access time from 2 to almost 1.

    Page Fault: Occurs when a program tries to access a page that is mapped in logical space but not loaded in RAM.

    EAT (Effective Access Time) Formula:
    EAT = (1 - p) * m + p * (page fault overhead)
    where p is page fault rate and m is memory access time.

    Page Replacement Algorithms

    • FIFO: Replace the oldest page. Suffers from Belady's Anomaly (More frames != Fewer faults).
    • LRU (Least Recently Used): Replace page not used for longest time. Most popular.
    • Optimal: Replace page that won't be used for the longest time in future. (Unrealizable).

    Thrashing: A state where the system spends more time swapping pages than executing instructions. Caused by high degree of multiprogramming and lack of RAM.

    Virtual Memory

    Execution of processes that are larger than physical RAM. It uses Demand Paging—load pages only when needed.

    6. File System & Allocation

    The File System provides a mechanism for online storage and access to data/programs on disk.

    File Allocation Methods

    MethodDescriptionPros/Cons
    ContiguousFile occupies consecutive blocks.Fast / External Fragmentation.
    LinkedEach block contains a pointer to the next.No Fragmentation / Slow seeking.
    IndexedAn index block stores pointers to all blocks.Random access / Index block overhead.

    Directory Structure

    Single-Level: All files in one directory (Naming conflicts).
    Two-Level: One folder per user.
    Tree-Structured: Hierarchical folders (Modern Standard).

    Inodes (Index Nodes): Data structures in Unix-like systems that store metadata about a file (Permissions, Owner, Size, Block Pointers) but not the filename.

    7. Disk Scheduling & I/O Management

    Goal: Minimize Seek Time

    SSTF: Shortest Seek Time First. Closer tracks first. Can cause starvation.
    SCAN (Elevator): Moves end-to-end, servicing requests in path.
    C-SCAN: One-way SCAN. Returns to start without servicing. More uniform wait time.
    LOOK: Like SCAN, but stops at the last request instead of end of disk.

    I/O Management Concepts

    • DMA (Direct Memory Access): Allows hardware to bypass CPU and transfer data directly to/from RAM. Frees CPU for other tasks.
    • Spooling: Stacking I/O jobs for a slow device (like a printer) in a buffer/disk.
    • Buffering: Storing data temporarily while it's being transferred between two devices/processes.

    8. Virtualization & Modern Concepts

    Virtual Machines (VMs)

    Hardware virtualization. Each VM runs a full guest OS. Heavy resource usage.
    Hypervisor: Software that creates and runs VMs (e.g., VMware, KVM).

    Containers (Docker)

    OS-level virtualization. Share the host kernel. Lightweight and fast. Portable across environments.

    Process & System Calls (C Snippets)

    fork() behavior

    Returns 0 to the child and the child's PID to the parent. Used to create concurrency.

    #include <stdio.h>
#include <unistd.h>

int main() {
    pid_t p = fork();
    if(p < 0) printf("Error");
    else if(p == 0) printf("Hello Child!");
    else printf("Parent, child is %d", p);
    return 0;
}
    Pthreads Introduction

    Lightweight processes that share memory. Crucial for high-performance computing.

    #include <pthread.h>
#include <stdio.h>

void* myTurn(void* arg) {
    printf("Thread Active...\n");
    return NULL;
}

int main() {
    pthread_t thread;
    pthread_create(&thread, NULL, myTurn, NULL);
    pthread_join(thread, NULL); 
    return 0;
}

    Placement Ready: Top 15 Interview Questions

    1. What is a Microkernel? (Essential services only in kernel, others in user-space).
    2. Process vs Thread? (Threads share memory, lighter weight).
    3. What is Thrashing? (Constant swapping due to insufficient RAM).
    4. Belady's Anomaly? (Increasing frames increases page faults in FIFO).
    5. Atomic Operations? (Operations that complete in one step without interruption).
    6. Context Switching Overhead? (Pure overhead, no work done while saving/loading states).
    7. Kernel vs User Mode? (Hardware protection via mode bits).
    1. Banker's Algorithm? (Used for Deadlock avoidance by simulating safe states).
    2. Starvation vs Deadlock? (Starvation is indefinite wait, Deadlock is circular wait).
    3. SPOOLing? (Simultaneous Peripheral Ops On-Line - bufferying slow device output).
    4. Hard vs Soft RTOS? (Hard has strict deadlines, Soft has priority).
    5. Internal vs External Fragmentation? (Space inside vs space between segments).
    6. Zombie Process? (Terminated but still has entry in process table).
    7. Orphan Process? (Child whose parent has crashed/terminated).
    8. What is aging? (Improving priority of long-waiting jobs to prevent starvation).
