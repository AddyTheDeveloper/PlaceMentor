    • Unit 1: Operating Systems

    The Core Bridge: Hardware Management, Process Control, and Memory Architecture.

    1. Foundations & Types of OS

    Technical Definition: An Operating System (OS) is a system software that manages computer hardware, software resources, and provides common services for computer programs.

    Architecture: Monolithic vs. Microkernel

    Monolithic Kernel

    All OS services (VFS, IPC, File System, Scheduler) run in kernel space. Fast, but one failure can crash the whole system.
    Examples: Linux, Windows.

    Microkernel

    Only essential services (IPC, Address Space) stay in kernel. Others move to user-space. Highly stable and secure.
    Examples: L4, QNX, Minix.

    2. Process Management & Life Cycle

    Process vs Program: A program is a 'passive' entity (code on disk). A process is an 'active' entity (code in execution).

    4. Process Synchronization & IPC

    Race Condition: A situation where several processes access and manipulate the same data concurrently and the outcome depends on the order of access.

    Critical Section Problem Requirements:

    • Mutual Exclusion: If process Pi is executing in its critical section, then no other processes can be executing in their critical sections.
    • Progress: If no process is executing in its critical section, the selection of the next process cannot be postponed indefinitely.
    • Bounded Waiting: There is a limit on the number of times other processes are allowed to enter their critical sections after a process has made a request.

    Question: Can we use RR for heavy computational tasks?
    Answer: No, frequent context switches would introduce high overhead. It's best for Time-Sharing systems.

    3. Deadlocks (The Infinite Wait)

    Deadlock: A situation where a set of processes are blocked because each process is holding a resource and waiting for another resource held by some other process.

    Necessary Conditions (Coffman Conditions)
    1. Mutual Exclusion: Resource can be held by only one process.
    2. Hold and Wait: Process holding resources can wait for more.
    3. No Preemption: Resource cannot be forcibly taken.
    4. Circular Wait: P1 waits for P2, P2 for P3... Pn for P1.
    Banker's Algorithm: Step-by-Step Example

    Scenario: 5 Processes (P0-P4), 3 Resources (A=10, B=5, C=7).

    ProcessAlloc (A B C)Max (A B C)Need (A B C)
    P00 1 07 5 37 4 3
    P12 0 03 2 21 2 2
    P23 0 29 0 26 0 0

    Step 1: Need = Max - Allocation (Calculated in table).

    Step 2: Check if Need ≤ Available. If yes, process finishes and releases Alloc.

    Step 3: Repeat until all finish or a deadlock is detected.

    Semaphores: An integer variable used for signaling. wait() decrements, signal() increments.

    IPC (Inter-Process Communication)

    • Shared Memory: Fastest, but needs sync.
    • Message Passing: Slower, but easy to implement and no sync needed.

    5. Memory Management

    Memory Management: The process of controlling and coordinating computer memory, assigning portions to various running programs to optimize system performance.

    Visual Learning: Memory Hierarchy

    Registers (Fastest, Smallest)
    L1/L2/L3 Cache
    Main Memory (RAM)
    Secondary Storage (HDD/SSD)

    Dynamic Loading: A routine is not loaded until it is called. | Dynamic Linking: Postpones linking until execution time.

    ConceptDefinition
    PagingMemory management scheme that eliminates the need for contiguous allocation of physical memory. (Non-contiguous)
    SegmentationMemory-management scheme that supports user view of memory (Logical segments like main, func, stack).
    FragmentationInternal: Space inside a block is wasted. External: Total free space is enough, but not contiguous.

    Page Table Entry (PTE) contains: Page Frame Number, Valid/Invalid bit, Dirty bit, Protection bits.

    Multi-Level Paging (The Solution to Large Page Tables)

    Flat page tables take too much RAM. Multi-level paging breaks the table into smaller pieces that are only brought into memory when needed.

    TLB (Translation Lookaside Buffer): A high-speed cache for page table entries.
    Effective Access Time = (Hit Rate * Cache Time) + (Miss Rate * Memory Time).

    Belady's Anomaly: For some page replacement algorithms (like FIFO), the page fault rate may increase as the number of allocated frames increases.

    Thrashing: A process is thrashing if it is spending more time paging than executing. Formula: CPU Utilization < (Time spent on Paging).

    Virtual Memory: Technique that allows execution of processes that are not completely in memory. It makes the computer appear to have more RAM than it actually does.

    3. Disk Scheduling

    Main goal: Minimize Seek Time (time to move disk arm).

    Placement Case Study: Thrashing

    Question: "How do you detect and fix thrashing in a system?"

    Answer: Thrashing is detected when CPU utilization drops while disk activity (paging) is at 100%.
    Step-by-Step Fix:

    1. Decrease the degree of multiprogramming (kill some processes).
    2. Increase physical RAM.
    3. Optimize page replacement algorithms.

    4. Multi-Threading with Pthreads

    Thread: A lightweight process. Multiple threads within a process share the same data and address space.

    #include <pthread.h>
#include <stdio.h>

void* myTurn(void* arg) {
    for(int i=0; i<8; i++) {
        sleep(1);
        printf("Thread Active...\n");
    }
    return NULL;
}

int main() {
    pthread_t thread;
    pthread_create(&thread, NULL, myTurn, NULL);
    pthread_join(thread, NULL); 
}

    5. Critical System Calls (C Snippets)

    fork(): Creates a child process. Returns 0 to child, PID of child to parent.

    exec(): Replaces the current process image with a new one.

    #include <stdio.h>
#include <unistd.h>

int main() {
    pid_t p = fork();
    if(p < 0) printf("Error");
    else if(p == 0) printf("I am Child");
    else printf("I am Parent, child is %d", p);
    return 0;
}
