    1. Fundamentals of Computer Networks

    Computer Network: A collection of autonomous computers interconnected by a single technology to share resources and exchange data.

    Types of Networks

    LAN (Local)

    Private networks within a single building or campus (up to few kms). High speed, low error rate.

    MAN (Metropolitan)

    Slightly larger area, e.g., an entire city. Often connects multiple LANs.

    WAN (Wide Area)

    Spans large geographical distances (countries/continents). The Internet is the largest WAN.

    Transmission Media

    Guided (Wired)
    • Twisted Pair: Ethernet (10/100/1000 Mbps). Cheap, easy to install.
    • Coaxial: TV Cables. High bandwidth, good immunity to noise.
    • Fiber Optic: Light signals. Extremely high speed, no EMI. Best for WAN/LAN backbones.
    Unguided (Wireless)
    • Radio Waves: Omnidirectional (WiFi, FM). Long distance.
    • Microwaves: Unidirectional (Satellite, Cell towers). Line-of-sight.
    • Infrared: Short range (TV Remote). Cannot pass through walls.

    Network Topologies

    TopologyWorking PrincipleBest For
    MeshEvery node connected to every other node. (n(n-1)/2 cables).High reliability (Military).
    StarAll nodes connected to a central Hub/Switch.Home/Office networks.
    BusSingle backbone cable (bus). Terminators at ends.Simple, old setups.
    RingEach node has two neighbors. Data flows in one direction.Token passing networks.
    Bandwidth, Throughput & Latency
    • Bandwidth: Max data transfer rate of a link (theoretical).
    • Throughput: Actual data transferred over time (practical).
    • Latency: Time taken for a packet to travel from source to destination.

    Transmission Modes

    Simplex

    One-way only (TV/Radio).

    Half-Duplex

    Both ways, but not at once (Walkie-Talkie).

    Full-Duplex

    Simultaneous two-way (Mobile Call).

    Switching Techniques

    Circuit Switching

    Dedicated path is established before data transfer (e.g., Landline). Pros: Guaranteed bandwidth. Cons: Resource wastage during silence.

    Packet Switching

    Data broken into packets; each packet can take a different route. Pros: Efficient, robust. Cons: Jitter/Delay variations.

    3. TCP/IP Model

    TCP (Transmission Control Protocol)
    • Connection-oriented.
    • Reliable (Retransmission).
    • Byte-stream oriented.
    • Flow & Congestion control.
    UDP (User Datagram Protocol)
    • Connectionless.
    • Unreliable (Best effort).
    • Message-oriented.
    • Fast (Low overhead).
    FeatureTCPUDP
    Header Size20 - 60 Bytes8 Bytes
    RetransmissionYes (if lost)No
    Flow ControlYes (Windowing)No
    Port AddressingProcess-to-processProcess-to-process

    TCP 3-Way Handshake (Connection Establishment)

    Client

    SYN (Seq=X)
    SYN+ACK (Seq=Y, Ack=X+1)
    ACK (Ack=Y+1)

    Server

    Why 3-way? To ensure both sides can send and receive. It prevents "Delayed Duplicate" packets from creating ghost connections.

    4. Data Link Layer

    Framing & MAC Addressing

    Framing: Breaking bitstream into manageable units called Frames. Uses methods like Character Count or Bit Stuffing.

    MAC Address: 48-bit (6 bytes) physical address. Example: 00:1A:2B:3C:4D:5E. First 3 bytes are OUI (Manufacturer ID).

    Multiple Access Protocols

    ALOHA (The Pioneer)
    • Pure ALOHA: Send whenever data is ready. Max Efficiency = 18.4%.
    • Slotted ALOHA: Time is divided into slots. Send only at start of slot. Max Efficiency = 36.8%.
    CSMA (Carrier Sense)
    • CSMA/CD: Collision Detection. Used in Ethernet (Wired). Stops sending as soon as collision is detected.
    • CSMA/CA: Collision Avoidance. Used in WiFi (Wireless). Uses RTS/CTS to avoid collisions.

    Framing: Bit Stuffing Example

    Pattern: 01111110 (Flag). If data has five 1s, add a 0.

    Input: 111111 → Output: 1111101 (Stuffing 0 after five 1s).

    Helps receiver identify where frame ends.

    Error Detection vs Correction

    Error Detection

    Finding if data was corrupted (Parity, Checksum, CRC). Only 1 bit needed usually.

    Error Correction

    Repairing corrupted data without retransmission (Hamming Code). Requires more redundant bits.

    Error Detection: CRC (Cyclic Redundancy Check)

    Numerical CRC Calculation Example

    Data: 1010, Generator (G): 101

    1. Append (G-1) zeros to data: 1010 + 00 = 101000
    2. Divide 101000 by 101 using XOR.
    3. Remainder + Data = Transmitted Frame.

    Flow Control

    • Stop-and-Wait: Efficiency = 1 / (1 + 2a). Safe but slow.
    • Sliding Window: Allows multiple frames in flight. (Go-Back-N, Selective Repeat).

    5. Network Layer (High Priority)

    IP Addressing (IPv4)

    C192 - 22324 / 8N.N.N.H
    IPv4 Header Fields
    • Version: Ver 4 or 6. (4 bits).
    • IHL: Internet Header Length (5 words = 20 bytes min).
    • TTL: Time To Live (Decremented by each router). Prevents infinite looping.
    • Protocol: Defines next layer (TCP=6, UDP=17).
    NAT (Network Address Translation)

    Allows multiple devices in a private network to share a single public IP. Useful for saving IPv4 address space.

    Subnetting Tip: Number of subnets = 2borrowed_bits. Number of hosts = 2host_bits - 2 (Minus Network ID and Broadcast ID).

    Routing Algorithms

    • Distance Vector (RIP): Bellman-Ford algorithm. Shares full table with neighbors. Problem: Count to Infinity.
    • Link State (OSPF): Dijkstra's algorithm. Shares local link state with everyone. Faster convergence.

    ARP (L2.5): IP → MAC.
    ICMP: Reporting errors (Ping/Traceroute).

    6. Transport Layer

    Congestion Control

    TCP handles congestion using 4 phases:

    1. Slow Start: Exponentially increase window size (1, 2, 4, 8...).
    2. Congestion Avoidance: Increase linearly after reaching threshold level.
    3. Fast Retransmit: Upon 3 duplicate ACKs.
    4. Fast Recovery: Skip slow start if some packets are getting through.

    Port Numbers: Well-known ports are 0-1023. Example: 80 (HTTP), 443 (HTTPS), 53 (DNS).

    7. Application Layer

    What happens when you type a URL? (Master Question)

    1. DNS Lookup: Browser checks cache → OS cache → Router cache → ISP (Recursive query to Root/TLD/Authoritative).
    2. TCP Handshake: Establishment of 3-way handshake with the server's IP.
    3. HTTP Request: Browser sends GET request.
    4. Server Response: Server sends 200 OK and index.html.
    5. Rendering: Browser parses HTML, CSS, and JS to show the page.
    HTTP vs HTTPS

    HTTPS = HTTP + SSL/TLS. Port 443. Encrypted and Authenticated.

    HTTP Versions (Interview Core)
    • HTTP 1.1: Persistent connections, Pipelining (Sequential).
    • HTTP 2: Multiplexing, Header compression (HPACK). Parallel requests over 1 TCP.
    • HTTP 3: Uses QUIC instead of TCP. Solves Head-of-line blocking.

    8. Network Security Basics

    Encryption Types

    • Symmetric: Same key for encryption and decryption (AES). Fast.
    • Asymmetric: Different keys (Public for encryption, Private for decryption). RSA.

    Common Attacks:
    - MiTM: Attacker sits between sender and receiver.
    - DDoS: Overwhelming a server with traffic from multiple sources.

    9. Important Numericals

    1. Delay Formula

    Total Delay = Propagation Delay + Transmission Delay + Queuing Delay + Processing Delay

    Transmission Delay = Packet Size / Bandwidth

    Propagation Delay = Distance / Speed

    2. Subnetting Problem

    Q: Given 192.168.1.0/26, find number of hosts and mask.

    A: /26 means 26 net bits, 6 host bits.
    Hosts = 26 - 2 = 62 usable hosts.
    Mask = 255.255.255.192

    3. Hamming Code (7, 4)

    Data: 1011. Find Hamming bits (P1, P2, P4).

    Logic: P1 covers 1, 3, 5, 7. P2 covers 2, 3, 6, 7. P4 covers 4, 5, 6, 7. Use Even Parity.

    A: P1=0, P2=1, P4=0. Codeword = 0110011.

    4. Routing: Dijkstra's Basic

    Q: Shortest path from A to C if A-B=2, B-C=3, A-C=6.

    A: Path A→B→C (2+3=5) is shorter than direct A→C (6).

    10. Placement FAQ: Master List

    1. 1. Difference between Hub, Switch, and Router?
      A Hub is at L1 (broadcasts everything). A Switch is at L2 (MAC-based, intelligent). A Router is at L3 (IP-based, connects different networks).
    2. 2. What is a Gateway?
      A node that serves as an entrance to another network, often used to connect a LAN to the Internet.
    3. 3. Explain the 3-Way Handshake.
      Client → SYN; Server → SYN-ACK; Client → ACK. It ensures reliable connection establishment.
    4. 4. How does DNS work?
      Converts human-readable names (google.com) to IP addresses using a hierarchical distributed database of Root, TLD, and Authoritative servers.
    5. 5. What is the difference between IPv4 and IPv6?
      IPv4 is 32-bit (numeric), IPv6 is 128-bit (hexadecimal). IPv6 provides nearly unlimited addresses and better security.
    6. 6. What is the difference between MAC and IP Address?
      MAC is physical (burned-in), unique for every device. IP is logical (assigned), unique for every device on a network.
    7. 7. What happens in a 4-way TCP termination?
      FIN → ACK; FIN → ACK. Ensures both sides are done sending data.
    8. 8. What is the use of TTL in IP Header?
      To prevent packets from looping infinitely. Each router decrements TTL; if it hits 0, packet is dropped.
    9. 9. How does HTTPS provide security?
      Uses SSL/TLS for Encryption (AES), Integrity (Hashing), and Authentication (Digital Certificates).
    10. 10. Difference between CSMA/CD and CSMA/CA?
      CD detects collisions (Ethernet). CA avoids collisions (WiFi) using RTS/CTS.
