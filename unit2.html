    OSI Model: A conceptual framework used to understand network interactions in 7 abstraction layers.

    Visual Learning: OSI Layer Data Flow

    Application Layer (Data)
    Transport Layer (Segments)
    Network Layer (Packets)
    Data Link Layer (Frames)
    Physical Layer (Bits)
    Layer 1: Physical (The Bit Stream)

    Multiplexing: Combining multiple signals into one medium.
    - FDM (Frequency): Radio/TV.
    - TDM (Time): Digital telephony.
    - WDM (Wavelength): Fiber optics.

    Layer 2: Data Link (Hop-to-Hop)

    Flow Control: Preventing sender from overwhelming receiver.
    - Stop-and-Wait: Sends 1 frame, waits for ACK.
    - Sliding Window: Sends multiple frames before needing ACK (Go-Back-N, Selective Repeat).

    Error Detection: CRC (Cyclic Redundancy Check) and Checksum.

    Subnetting: The process of dividing a large network into smaller, manageable sub-networks (subnets). This helps in reducing network traffic and enhancing security.

    ARP and ICMP (Helper Protocols)
    • ARP (Address Resolution Protocol): Maps IP address to MAC address.
    • ICMP (Internet Control Message Protocol): Used for error reporting (e.g., Ping, Traceroute).

    IPv4 vs IPv6:
    IPv4: 32-bit, decimal notation, 4.3 billion addresses.
    IPv6: 128-bit, hexadecimal notation, virtually unlimited addresses (340 undecillion).

    AspectIPv4IPv6
    Address Space32-bit (4 Billion)128-bit (Infinity...)
    NotationDotted DecimalHexadecimal
    HeaderVariable (20-60 bytes)Fixed (40 bytes)
    Routing Protocols: DV vs Link State

    Distance Vector (RIP): Each router tells its neighbors about the whole world. Based on Bellman-Ford.
    Link State (OSPF): Each router tells the whole world about its neighbors. Based on Dijkstra.

    Application Layer Protocols:
    - HTTP/HTTPS: Port 80/443.
    - DNS: Port 53 (UDP).
    - DHCP: Dynamic IP allocation.

    TCP/IP vs OSI: OSI is a theoretical model; TCP/IP is the practical implementation used on the Internet. TCP/IP has 4 layers: Application, Transport, Internet, Network Access.

    4. Error Detection: CRC & Checksum

    CRC (Cyclic Redundancy Check): Uses polynomial division to detect errors. A generator polynomial is used to create a checksum at the sender, and the receiver verifies it.

    5. Congestion Control

    Leaky Bucket

    Output rate is fixed. Regardless of bursty input, packets flow out at a constant pace.

    Token Bucket

    Allows bursty output if tokens are available. More flexible than Leaky Bucket.

    IP Subnetting: "How to Solve" Guide

    Problem: Divide 192.168.1.0/24 into 4 subnets.

    1. Find Bits Needed: 2^n ≥ 4 subnets → n = 2 bits.
    2. New Mask: /24 + 2 = /26 (255.255.255.192).
    3. Block Size: 256 - 192 = 64.
    4. Subnet Ranges:
      1: 192.168.1.0 to .63
      2: 192.168.1.64 to .127
      3: 192.168.1.128 to .191
      4: 192.168.1.192 to .255

    Placement Case Study: 3-Way Handshake

    Question: "Why is a 2-way handshake insufficient for TCP?"

    Answer: To prevent "half-open" connections. A 3-way handshake ensures that both sides have received a sequence number and are ready to communicate.
    Sequence: SYN(S) → SYN(R)+ACK(S) → ACK(R).
