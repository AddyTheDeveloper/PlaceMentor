    Advanced C++: Smart Pointers (Memory Safety)
    • std::unique_ptr: Exclusive ownership. Non-copyable.
    • std::shared_ptr: Reference counted. Shared ownership.
    • std::weak_ptr: Non-owning reference to avoid circular dependencies.

    SOLID Principles (Job Interview Favorite):
    - S: Single Responsibility.
    - O: Open/Closed (Open for extension, closed for modification).
    - L: Liskov Substitution.
    - I: Interface Segregation.
    - D: Dependency Inversion.

    Pointer: A variable that stores the physical memory address of another variable.

    Crucial Pointer Types (Interview Favs)

    • NULL Pointer: Points to nothing. int *p = NULL;
    • Void Pointer: Generic pointer, can point to any type. Needs typecasting to dereference.
    • Wild Pointer: Uninitialized pointer. DANGEROUS!
    • Dangling Pointer: Points to a memory location that has been freed/deleted.

    Dereferencing (*): Accessing the data at the address held by the pointer.

    Address-Of (&): Getting the memory address of a variable.

    6. Runtime Polymorphism: "How it Works"

    Scenario: Base* ptr = new Derived(); ptr->show();

    1. V-Table: Every class with a virtual function has a Static V-Table containing function pointers.
    2. V-Pointer (vptr): Every object of such a class has a hidden pointer to its class's V-Table.
    3. Resolution: At runtime, the code follows ptr -> vptr -> V-Table -> Derived::show().

    Interview Tip: Mention that Virtual Functions increase the object size by the size of one pointer (vptr).

    3. Parameter Passing Techniques

    Call by Value

    Copy made. Original unaffected.

    void swap(int x, int y)

    Call by Reference (C++)

    Alias created. Original changes.

    void swap(int &x, int &y)

    Call by Address

    Address passed. Dereference to change.

    void swap(int *x, int *y)
    Stack View:
    [ Main: a=5 ] ← [ Swap: ref to a ] (By Reference/Address)
    Placement Case Study: The Diamond Problem

    Question: "How does C++ solve the diamond problem in multiple inheritance?"

    Answer: By using virtual inheritance. This ensures that only one instance of the base class is inherited by the final subclass.
    Syntax: class A : virtual public Base { ... };

    Virtual Functions vs Abstract Classes (Pure Virtual):
    Virtual Function: Function in base class that can be overridden.
    Pure Virtual: virtual void func() = 0; has no implementation. Makes class Abstract (cannot instantiate).
