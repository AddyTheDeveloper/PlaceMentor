    1. Asymptotic Analysis (The Math of Code)

    Master Theorem: T(n) = aT(n/b) + f(n)

    • Case 1: If f(n) < n^(log_b a), then T(n) = Θ(n^(log_b a)).
    • Case 2: If f(n) = n^(log_b a), then T(n) = Θ(n^(log_b a) * log n).
    • Case 3: If f(n) > n^(log_b a), then T(n) = Θ(f(n)).
    NotationMeaningRate of Growth
    O(1)ConstantFastest. Doesn't depend on input.
    O(log n)LogarithmicVery fast. (e.g., Binary Search).
    O(n)LinearProportional to input size.
    O(n log n)Log-linearEfficient sorting (e.g., Merge Sort).
    O(n²)QuadraticSlower. (e.g., Bubble Sort).

    3. Advanced Paradigms: DP vs Greedy

    Dynamic Programming

    • Solves sub-problems once.
    • Memoization: Top-down (Recursion).
    • Tabulation: Bottom-up (Iteration).
    • Example: Knapsack, LCS.

    Greedy Strategy

    • Makes the local optimal choice.
    • Doesn't re-examine choices.
    • Fast but not always optimal.
    • Example: Huffman, Dijkstra.

    4. Shortest Paths: Dijkstra Trace

    Goal: Find shortest path from Source S to all nodes.

    1. Init: Dist[S]=0, Dist[Others]=∞. Min-Priority Queue PQ holds (0, S).
    2. Step: Pop (d, u) from PQ.
    3. Relax: For each neighbor v of u: if Dist[u] + weight(u,v) < Dist[v], then update Dist[v] and push to PQ.
    4. Repeat until PQ is empty.

    Complexity: O((V+E) log V).

    5. KMP String Matching

    P: Solvable in Polynomial time (e.g., Sorting).

    NP: Verifiable in Polynomial time (e.g., Sudoku).

    NP-Hard: At least as hard as the hardest problems in NP.

    NP-Complete: Problems that are both in NP and NP-Hard (e.g., TSP, Knapsack).

    Placement Case Study: DP vs Greedy

    Question: "Can the 0/1 Knapsack problem be solved optimally with a Greedy approach?"

    Answer: No. Greedy only works for the Fractional Knapsack problem. For 0/1, you must use Dynamic Programming to ensure you find the global maximum value.

    Longest Common Subsequence (LCS) Trace

    To find LCS of "ABCDE" and "ACE":
    1. If chars match: 1 + LCS(rest)
    2. If chars mismatch: Max(LCS(s1, rest2), LCS(rest1, s2))
    Time: O(m*n) with DP. Space: O(m*n).

    Rabin-Karp Algorithm: String matching using Hashing. Rolling hash allows O(n+m) average complexity.
